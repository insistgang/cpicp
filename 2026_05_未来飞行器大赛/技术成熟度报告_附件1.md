## 2. 技术底座详细说明

### 2.1 目标检测算法（TRL 7）

**技术来源**：复用 2026_03 智能建造赛 / 2026_04 AI+X 赛技术成果

**核心能力**：
- 模型：YOLOv12n + P2 小目标检测头
- 输入分辨率：1024x1024
- 检测类别：建筑、屋顶加建、临时棚屋、施工机械、堆料区
- 标签分配：NWD（归一化 Wasserstein 距离）
- 损失函数：Wise-IoU / Inner-IoU
- 时序滤波：连续帧确认机制

**验证状态**：
- [x] 公开数据集验证（COCO、自定义违建数据集）
- [x] 边缘端推理验证（Jetson Orin Nano，≥30FPS）
- [ ] 真实场景飞行验证（待完成）

**局限与风险**：
- 真实场景光照、天气、遮挡条件复杂，需现场数据微调
- 不同城市建筑风格差异大，需域适配

### 2.2 地理定位系统（TRL 6）

**技术来源**：复用 geolocate 模块

**核心能力**：
- 输入：检测框像素坐标 + 相机内参 + POS 数据（GPS/IMU）+ DEM
- 输出：WGS-84 地理坐标（经纬度+高程）
- 方法：针孔相机模型 + 射线-地面求交

**精度评估**：
| 条件 | 平面误差 | 高程误差 |
|-----|---------|---------|
| 理想条件（平坦地形、精确 POS） | 1-2m | 2-3m |
| 一般条件（丘陵地形、消费级 GPS） | 3-5m | 5-10m |
| 复杂条件（高楼遮挡、多路径） | 5-10m | 10-20m |

**验证状态**：
- [x] 模拟数据验证
- [x] 地面控制点校验
- [ ] 大规模飞行验证（待完成）

### 2.3 时序跟踪滤波（TRL 7）

**技术来源**：复用 track_filter 模块

**核心能力**：
- 多目标跟踪：DeepSORT / ByteTrack 改进版
- 时序一致性：连续 N 帧确认机制（N=3-5），降低单帧误报
- 轨迹关联：IOU + 外观特征融合

**验证状态**：
- [x] 视频序列验证
- [x] 多场景测试（城市、郊区、水域）
- [x] 误报抑制率：反光/阴影误报降低 60-70%

### 2.4 变化检测（TRL 5-6）

**技术方案**：
- 方法：基于深度学习的语义变化检测（Change Detection）
- 输入：多时相遥感影像（历史 vs 当前）
- 输出：变化图斑 + 变化类型分类

**验证状态**：
- [x] 公开数据集验证（LEVIR-CD、WHU-CD）
- [ ] 真实城市数据验证（待完成）
- [ ] 与规划数据库对接验证（待完成）

**局限**：
- 需要高质量历史影像数据
- 季节性变化（植被、光照）需智能过滤

## 3. 技术风险与缓解措施

| 风险项 | 等级 | 影响 | 缓解措施 |
|--------|------|------|---------|
| 真实场景检测精度下降 | 中 | 误报/漏报增加 | 现场数据收集+微调；人机协同审核 |
| 定位精度不足 | 中 | 执法取证效力降低 | 结合 RTK/PPK 提升精度；多源数据融合 |
| 飞行器续航限制 | 低 | 单次覆盖面积有限 | 多架次协同；充电/换电机制 |
| 空域审批周期 | 中 | 响应时效性降低 | 提前申请常态化空域；自动化审批流程 |
| 数据安全合规 | 低 | 法律风险 | 加密传输、脱敏处理、权限管控 |
| 天气条件限制 | 低 | 作业窗口受限 | 气象预报集成；弹性任务调度 |

## 5. 与现有技术对比
