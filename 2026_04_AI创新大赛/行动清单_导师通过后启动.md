## 推荐主线

优先主攻线A：华为 AOI 少样本异常检测。

原因：

1. 任务边界清楚：输入图片/视频，输出正常/异常判定和缺陷定位。
2. 官方少样本设定适合 PatchCore/PaDiM：100 张正常 + 30 张缺陷迁移，再测 1000+。
3. 代码已有完整 baseline：合成 AOI 图、特征后端、memory bank、阈值、指标、热力图、在线学习闭环。
4. 本地 14 模块自测已全部通过，当前工程基础比 03 更完整。
5. AI+X 违建线可以作为低成本第二投，复用 03 的时序滤波和地理定位模块。

## GPU 上机动作

在 `src` 目录执行：

```bash
bash setup_gpu.sh
```

单独验证时执行：

```bash
python run_real_pipeline.py --real
python bench_latency_gpu.py --real --size 2500 --runs 20
```

需要记录：

1. `is_real_feature` 是否为 `true`。
2. AUC、F1、Recall、Precision、per-class recall。
3. 2500x2500 单图总延时是否满足 GPU `<200ms@2060` 红线。
4. 如果 wide_resnet50_2 太慢，改用 `MODEL=resnet18 bash setup_gpu.sh` 复测。

## 数据到手后的技术动作

1. 用 `aoi_prepare.py` 把真实数据转换为 manifest。
2. 复刻官方少样本协议：100 正常 + 30 缺陷迁移，测试 1000+。
3. 跑经典 CPU baseline，确认数据读取、阈值、指标、热力图全链路正确。
4. 跑 timm 真特征，比较 ClassicBackend、ResNet18、WideResNet50 的准确率和延时。
5. 跑在线学习闭环：误检样本入库、漏检反馈、阈值重标定，输出反馈次数到 F1 的变化曲线。
6. 生成 PPT 可用图：热力叠加、异常分布、ROC/PR、per-class 检出、延时对比。
