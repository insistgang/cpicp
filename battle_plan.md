## 比赛 1:网络安全(~8 月中)

### 基本信息
- **赛事**:中国研究生网络安全创新大赛
- **赛道**:揭榜挑战赛 - 题目 #4
- **赛题**:通用 AI Agent 的安全风险检测与防御闭环
- **去年成绩**:国三 + 差点国二(同方向迭代)
- **队伍上限**:1-4 人 + 1 名指导教师(**必需**)
- **奖金**:一等奖 1.5 万 / 二等奖 0.8 万

### 战略定位
**主攻,冲国一。** 去年国家级专家认可的方案 + 临门一脚的差距。今年补足短板,从国三直接冲国一。

### 核心策略:复用 + 升级

**不重写,只加层。** 去年代码 70-100% 可复用,实际增量工作量 1.5-2 周。

```
去年方案(国三基线)
    │
    ├─ 加层 B: 自适应防御闭环
    │   └─ 检测后自动生成防御策略,反馈 Agent 系统
    │
    └─ 加层 C: 真实部署 Demo
        └─ 带 GUI 的攻防演示平台
```

### 主推方向:B + C 组合

**B. 自适应防御闭环**
- 不只检测后报警,**检测到攻击后自动生成防御策略**(规则/约束/拦截器)
- 反馈到 Agent 系统,真正实现"**闭环**"二字
- 这正是题目标题"检测与防御闭环"的核心题眼

**C. 真实部署 Demo**
- 带 GUI 的 Agent 攻防演示平台
- 现场演示:红队攻击 → 系统检测 → 自动防御 → 攻击失败
- 答辩杀手锏,弥补去年可能 demo 较弱的短板

### 备选方向(根据去年情况选择)
- **A. MCP 协议层扩展**:借 2026 年 MCP 热度,蹭话题
- **D. 多 Agent 协同防御**:对接你 LangGraph orchestration 探索
- **E. 量化评估 Benchmark**:论文友好,可独立投学术会议

### 时间预算
- 7/12 - 7/20:整理去年代码 + 升级方案设计
- 7/21 - 7/31:新模块开发(B 闭环 + C Demo)
- 8/1 - 8/10:实验完成 + 技术报告
- 8/10 - 8/15:答辩 PPT + Demo 录屏
- 8 月中:提交

### 战略要点
- ⚠️ **必须找回去年所有材料**:代码、数据、PPT、答辩录像、专家反馈
- ⚠️ **必须确认指导教师**:去年导师今年是否还能挂

## 比赛 2:金融科技(8/17)

### 基本信息
- **赛道**:揭榜挂帅 - 赛题 #23
- **赛题**:基于多模态大模型的金融影像智能相似度检测模型
- **数据**:命题方提供 ~1 万张金融票据、证照影像
- **队伍上限**:6 人
- **联系**:南京大学 苏老师 `sutong@nju.edu.cn`

### 技术路线

**双路融合架构:**

```
        [输入: 票据 / 证照影像]
              │
       ┌──────┴──────┐
       ▼             ▼
   MLLM 上路        细粒度下路
  (Qwen2.5-VL)    (DINOv3 + 篡改感知头)
       │             │
   版式/字段/印章    像素级篡改定位
   语义理解         + 局部特征
       │             │
       └──────┬──────┘
              ▼
        融合决策头
              │
    [相似度, 真伪概率, 热力图, 文字解释]
```

### 创新点(精益画布用)
1. MLLM 语义 + 像素细粒度的**双路联合学习**——首次将多模态大模型用于金融文档真伪联合判定
2. 相似度 + 真伪 + 篡改定位**三任务共享 backbone**
3. **可解释输出**:篡改热力图 + 自然语言解释
4. **AIGC 鲁棒性专项**:Diffusion 时代金融文档篡改基准

### 关键指标
- 真伪二分类 AUC ≥ 0.95
- 篡改区域 IoU ≥ 0.7
- 相似度 Top-1 准确率 ≥ 90%
- 单图推理 ≤ 2 秒
- AIGC 伪造识别率 ≥ 85%

### 技术栈
| 组件 | 选型 |
|---|---|
| MLLM 主干 | Qwen2.5-VL-7B |
| 视觉细粒度 | DINOv3 / ViT-L |
| 篡改定位 | 复用现有网络 |
| 微调 | LoRA + QLoRA |
| Demo | Gradio |
| 算力 | 北京 4090 服务器 |

### 提交材料(4 项)
1. 精益画布 PPT(官方模板)
2. 可运行项目成果(代码 + 模型 + Demo 视频)
3. 技术文档 PDF(8-12 页)
4. 原创性声明(官方模板,签字扫描)

### 战略要点
- ⚠️ 跑数据前必须签《数据资料保密承诺书》,发邮件给苏老师
- ⚠️ 作品中不得出现任何学校 logo/校名/校徽
- ✅ 同一份代码可同步投 CAS Q2/Q3 期刊(IEEE TIFS / Pattern Recognition Letters / ESWA)
