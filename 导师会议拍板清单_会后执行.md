# 导师会议拍板清单与会后执行

用途：开会时逐项确认，避免会后方向含糊。

## 会上必须拍板

| 编号 | 问题 | 推荐答案 | 影响 |
|---|---|---|---|
| 1 | 01 网络安全是否主攻？ | 能找回去年材料则主攻 | 决定是否投入安全 demo 和答辩材料 |
| 2 | 02 金融科技是否立即申请数据？ | 是 | 报名/提交较早，越晚越被动 |
| 3 | 04 AI 创新是否组 3 人队伍？ | 是，作为强备选 | 决定是否走 chaspark 和 GPU 真特征验证 |
| 4 | 03 智能建造是否暂缓？ | 是，等数据和 Jetson 条件确认 | 避免过早消耗训练/部署时间 |
| 5 | 05 未来飞行器是否副线报名？ | 可报，但报告负责人另设 | 不占用算法主力 |
| 6 | 06 文化中国是否副线报名？ | 可报，但文史负责人主导 | 不占用算法主力 |

## 会后 24 小时内

1. 确认 01 去年材料是否存在。
2. 发出 02 数据申请邮件。
3. 确认 04 队长、队员和 chaspark 账号。
4. 把 03 的数据申请联系人、Jetson 设备状态列成单独表。
5. 给 05/06 指定报告/PPT 负责人。

## 会后 3 天内

1. 01：把去年代码、PPT、答辩录像、专家反馈放入私有资料夹。
2. 02：确认保密承诺书流程，准备真实数据目录结构。
3. 04：准备 GPU 环境，跑 `run_real_pipeline.py --real` 和 `bench_latency_gpu.py --real`。
4. 03：如果老师批准，发数据申请；否则保持暂停。
5. 05：补充成本收益、飞行器参数、低空政策素材。
6. 06：补充史料来源，优化 5 分钟讲解稿。

## 会后 1 周内

| 赛题 | 一周内可交付 |
|---|---|
| 01 | 去年材料复用评估 + 新闭环 demo 录屏方案 |
| 02 | 数据申请状态 + baseline 复测报告 |
| 03 | 数据申请状态 + Jetson 推理验证计划 |
| 04 | GPU 真特征指标 + 延时验证结果 |
| 05 | 项目报告书修订版 + 附件清单 |
| 06 | PPTX 二稿 + 语音讲解稿 |

## 风险红线

1. 不要同时把 01、02、03、04 都当主攻。
2. 03 没有数据和 Jetson 验证前，不投入重训练。
3. 06 不要做成 OCR/翻译系统，赛题重点不是技术系统。
4. 05 不要按 mAP 或算法比赛写，评分核心是应用场景和落地方案。
5. 所有提交物必须做匿名检查。

## 建议最终组合

默认组合：

- 主攻：01 或 02 二选一。
- 强备选：04。
- 条件推进：03。
- 副线：05、06。

如果 01 去年材料找不回：

- 主攻：02。
- 强备选：04。
- 保留：03。
- 副线：05、06。

如果 02 数据申请不顺：

- 主攻：04。
- 01 视去年材料决定是否回补。
- 03 等数据。
- 05/06 继续低成本推进。
