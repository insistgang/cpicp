# 赛事 D · 华为杯第二十三届中国研究生数学建模竞赛（轻量简报）

> 适用：CV/AI 博士目标(Python/机器学习/优化，RTX 4070 Super)。本届固定命题，赛题赛时公布(报名截止约 9/19，论文提交约 9/27-28，赛程一般 4 天)。基于公开历年资料，不预测本届题目。

## 1. 赛题要点（固定命题=赛时公布；历年题型概览）

研赛每年出 **A-F 共 6 题**任选一，赛程通常 4 天。6 题覆盖：工程/通信优化、数据统计建模与预测、机器学习/AI、信号与图像处理、运筹排班调度、生物医学建模。近年明显**偏数据驱动与 AI**(图像分割、深度学习、迁移学习、故障诊断、临近预报)。

| 年份 | 题目方向 | 类型 |
|---|---|---|
| 2025(22届) | A NPU 调度 / B 通信链路(LightGBM) / C 地质裂隙识别(U-Net+三维重构) / D 低空湍流监测 / E 轴承故障诊断(迁移学习) / F 园林美学量化 | **多题强 AI 倾向** |
| 2024(21届) | 风电场功率优化 / WLAN 吞吐量 / 磁性元件损耗 / 大数据地理综合 / 高速应急车道 / X射线脉冲星 | 数据+工程 |
| 2023(20届) | A WLAN 信道接入 / B DFT 矩阵整数分解 / C 竞赛评审方案 / D 双碳路径规划 / E 出血性脑卒中智能诊疗 / F 强对流降水临近预报 | C/E/F 含数据建模 |
| 2022(19届) | A 超分辨定位 / B 方形件组批 / C 涂装-总装缓存调序 / D PISA 芯片资源 / E 草原放牧 / F COVID 物资管理 | 运筹偏多 |
| 2021(18届) | A 相关矩阵低复杂度 / B 空气质量二次建模 / C 帕金森脑深部电刺激 / D 抗胰腺癌药物 / E UWB 定位 / F 航空机组排班 | 数据+医学+运筹 |

**对 CV/AI 用户判断**：几乎每年有 1-3 道可直接发挥编程/ML 优势的题(图像分割、信号分类、预测建模、智能诊疗)，占比逐年上升。

## 2. 评分/评审标准

官方公开评审主维度：**"假设的合理性、建模的创造性、结果的正确性、文字表述的清晰度"** 四项。流程：专家评分标准化求和排序、去最高最低、分歧大时协商。获奖比例：一等奖一般 ≤**1.5%**，二等约 **13%**，三等约 **20%**；另设华为专项奖、数模之星。确切分项权重官方未逐条公开。来源：[评审标准与获奖比例(西南交大)](https://maths.swjtu.edu.cn/info/1150/21154.htm)、[评审方案研究(知乎)](https://zhuanlan.zhihu.com/p/657725524)。

## 3. 技术路线（常用方法与工具库）

- **优化/运筹**：Gurobi(学生免费，首选)、CVXPY(凸优化)、`scipy.optimize`、`PuLP`/`OR-Tools`(整数/调度)；启发式 `DEAP`、`pymoo`(NSGA-II 多目标)。
- **机器学习/AI**：`scikit-learn`(基线)、`xgboost`/`lightgbm`(表格夺分主力)、`PyTorch`(深度/图像分割，4070S 12GB 可跑中等模型)；CV 题 `segmentation-models-pytorch`(U-Net)、`opencv-python`、`albumentations`。
- **统计/预测**：`statsmodels`、`prophet`、LSTM/Transformer；`pandas`+`numpy`。
- **评价类方法**：AHP、熵权法、TOPSIS、灰色关联、PCA(指标体系题)。
- **论文**：LaTeX(`ctex` 中文模板，Overleaf)；`matplotlib`/`seaborn`；流程图 draw.io。

## 4. 对照我的背景（选数据/AI 类题的优势）

你的 CV/AI+Python+优化与近年题型高度契合：
- **图像/信号题**(2025C 地质裂隙 U-Net、2023F 降水临近预报、2025E 轴承故障)直接吃 PyTorch+CV 功底，多数队伍卡在"建不出像样深度模型"，这是差异化得分点。
- **ML 预测/诊疗题**(2023E 脑卒中、2021B 空气质量)需特征工程+xgboost/lightgbm+评估，是你强项。
- **优化题**也能驾驭(Gurobi/CVXPY)，但与统计/运筹背景队伍竞争激烈，作备选而非首选。

结论：**优先选含图像/深度学习/数据预测的题**，把"能真正跑出有效 AI 模型+严谨实验对比"作为创新与正确性双重得分点。

## 5. 工作量粗估

- **赛前准备(人均 2-4 周，每周 5-8h)**：跑通往年 1-2 道 AI/数据题完整流程；搭好 PyTorch/Gurobi/LaTeX；固化代码模板(数据加载/交叉验证/画图/导出)与论文 LaTeX 骨架。
- **竞赛期(4 天，3 人)**：Day1 读题选题+建模框架；Day2-3 建模与编程(你主导 AI)，建模手推公式，写作手同步起草；Day4 上午补实验/敏感性、出图、定稿排版、提交。论文 20-40 页，**留足最后半天排版**。
- **团队/硬件口径**：研赛本就是固定 3 人赛(无需"人·周→日历周"换算)；**硬件仅需 RTX4070S 跑中小模型，无任何国产平台/边缘部署需求**——五赛中硬件门槛最低。

## 6. 推荐方案与主要风险

**推荐**：3 人分工 = 建模手(假设/公式) + 编程手=你(AI/数据实现/出图) + 写作手(LaTeX/逻辑/摘要)。选题：开赛 2-4 小时内锁定"含图像/深度学习/数据预测 + 数据可获取"的题，避开纯数值/纯运筹(若数学短板)。取胜点：模型合理性+结果正确+AI 创新+论文清晰，尤其补"假设合理性""敏感性/鲁棒性分析"(AI 队伍常忽略)。

**主要风险**：偏科(只堆 DL 缺数学严谨性会扣"建模创造性")；论文表达弱(代码强但写作弱是常见失分，写作手全程并行)；算力/数据(4070S 够中小模型，大数据需降采样)；选题优柔寡断(决策不超过半天)。
