# 导师审批汇报稿：六个赛题选择建议

汇报目的：请指导老师确认主攻赛题、备选赛题和低成本副线，并批准数据申请、队伍组织和后续投入方向。

## 一句话结论

建议采用“1 个主攻 + 1 个强备选 + 2 个低成本副线”的策略：

- **优先主攻**：01 网络安全，前提是能找回去年作品和专家反馈。
- **稳妥主攻备选**：02 金融科技，工程链路最稳，需尽快申请数据。
- **强备选**：04 AI 创新，技术基础完整，但受 3 人队伍、chaspark 数据和 GPU 真特征验证约束。
- **暂缓条件推进**：03 智能建造，需数据、真实视频和 Jetson Orin Nano 端侧验证。
- **低成本副线**：05 未来飞行器、06 文化中国，主要是报告/PPT 型作品，不占用主力算力。

## 推荐拍板顺序

### 1. 先拍 01 是否主攻

01 的优势最大：去年已有国三基础，今年已有“红队攻击 -> 检测 -> 自动防御 -> 拦截复测”的闭环 demo。  
但它的核心前提是必须找回去年材料，否则“已有积累”无法转化为真实竞争力。

请老师拍板：

- 是否继续以 01 作为冲奖主攻。
- 去年指导教师今年是否还能挂靠。
- 是否允许先建立私有目录整理去年代码、PPT、答辩录像和专家反馈。

### 2. 若 01 材料找不回，主攻切到 02

02 是当前最稳的工程主线：赛题边界清楚、baseline 已跑通、硬件风险低。  
主要卡点是数据必须签保密承诺书后申请，且报名/提交时间较早。

请老师拍板：

- 是否批准申请金融科技赛题数据。
- 是否由学校/团队签署数据资料保密承诺书。
- 是否将 02 作为稳妥主攻或保底主攻。

### 3. 04 作为强备选

04 的 AOI 少样本异常检测已经具备完整端到端 baseline，适合算法展示和系统 demo。  
主要限制是队伍上限 3 人、需要华为 chaspark 数据、需要 GPU 上验证真特征和延时红线。

请老师拍板：

- 是否组 3 人小队报名。
- 是否走华为企业赛题 AOI，或 AOI + AI+X 双投。
- 是否安排一台 GPU 机器跑真特征和延时验证。

### 4. 03 暂缓，等数据和硬件条件确认

03 可以做，但比 02/04 更吃真实数据、真实视频和 Jetson Orin Nano 端侧工程。  
新款 Jetson Orin Nano 8G 可以做推理验证，但训练仍需 4070S/4090 等 GPU。

请老师拍板：

- 是否批准向命题方申请数据。
- 是否确认 Jetson Orin Nano 8G 作为端侧验证设备。
- 是否接受训练和部署拆分：GPU 训练，Jetson 推理。

### 5. 05/06 作为低成本副线

05 和 06 都不是技术主攻型赛题。  
05 是低空经济应用场景报告书，06 是文化内涵/PPT 展示作品。它们适合在不占用主力算力的情况下走流程。

请老师拍板：

- 是否允许作为副线报名。
- 是否能补充跨学科或文史方向队员。
- 是否由非技术主力负责报告、PPT、讲稿和匿名合规。
