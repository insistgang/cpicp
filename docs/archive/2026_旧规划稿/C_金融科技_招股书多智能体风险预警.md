## 1. 赛题要点（**来源：赛题书附件**，已覆盖上轮"未获取"）

- **命题单位**：东吴证券股份有限公司（苏州工业园区）。联系人 任川（信息技术总部副总经理），rench@dwzq.com.cn / 18610756988。**所属子赛道：揭榜挂帅赛道二「智能风控与量化建模」**（全大赛共 27 道揭榜题、4 子赛道，每题配专用数据集）。
- **命题概述**：针对港股上市企业（**尤其未盈利的生物科技/特专科技公司**）招股书专业度高、披露复杂的痛点，基于 LLM + **Agentic AI** 构建含「**基本面风控**」与「**市场情绪**」的多角色智能体系统；交叉预警"内在财务/合规暗雷"与"外在流动性"，并用**上市当天/一周/一个月/三个月内真实跌幅与破发数据**反向验证。
- **三大痛点（赛题书原文）**：① 招股书数百页、估值对赌/大额关联交易/现金流消耗等隐性暗雷难人工排查；② 传统预警视角单一，缺法务合规+财务穿透+市场博弈多角色协同；③ 缺"基本面+市场情绪"共振评估（破发常受发行期大盘遇冷/板块流动性枯竭影响）。
- **三项技术攻关问题（赛题书原文）**：
  1. LLM **防幻觉**抽取标准化财务指标 + 非标隐性风险（**现金消耗率、对赌/赎回条款、关联交易、客户/供应商集中度、核心管线进度**），并识别"风险因素"章节的**文本粉饰度**；
  2. Agentic AI 下设计**法务合规 / 财务穿透 / 市场分析**多专家角色协作架构，冲突时通过**自主规划、辩论、查证链路**实现交叉验证与风险判定；
  3. **跨模态融合**：低频静态文本特征（招股书瑕疵）× 高频动态时序特征（大盘冷暖、板块流动性）。
- **命题方现状（赛题书）**：东吴已有以"反思"为核心的多智能体架构(合规/审计/投研巡检问答)经验；已沉淀**近 3 年港股招股书、港股基本信息、历史行情**数据。

## 2. 评分/评审标准 与 交付要求（**来源：赛题书附件 + 参赛指南**，已覆盖上轮"未获取"）

**赛题书给出的硬技术指标（验收口径）**：
- 招股书关键风险要素**抽取准确率 ≥ 80%**（对赌/赎回、关联交易、客户/供应商集中度、现金流消耗等非标风险）；
- 关键**证据片段召回率 ≥ 85%**（能从原文准确定位支撑结论的段落/表格）；
- Agent 推理链路 / 角色分工 / 工具调用 / 证据来源**可追踪率 100%**，逻辑解释有效性由专家或 LLM 辅助评审打分；
- 业务参考价值：用上市**首日 / 5 / 20 / 60 交易日**真实表现评估，**重点考核"上市后 5 个交易日内显著下跌风险"识别能力，并给予更高权重**。

**成果交付（赛题书）**：① 完整项目源码（含环境/运行脚本）**或**主流 Agentic AI 框架中可复用的 IPO 风险预警 **Skill**；② **可运行原型/API**（输入公司名/股票代码/招股书 → 输出预警提示+报告，支持投研人机协同复核）；③ 测试集预测结果表 + 多智能体推理日志 + 关键证据片段 + 典型案例分析报告。

**评审流程（参赛指南）**：初赛 = **统一可运行性测试 + 线上评审**（揭榜与自拟分开评审；**无法正常运行将影响评审**）；决赛 = **全国总决赛现场路演及答辩**（用组委会统一 PPT 模版）。初赛须交 4 项材料（**含"作品原创性声明"，缺则不具备通过初赛资格**）；决赛交 2 项材料。
**评分分项数值权重**：参赛指南/赛题书未见明确百分比，**此项仍标"未获取"**（仅知"5 日内下跌识别"权重更高）。

## 3. 技术路线（仍有效，本轮未改；与赛题书三攻关点对齐）

> 赛题书三攻关点 ↔ 下文：攻关1=①解析+防幻觉抽取，攻关2=②多智能体协同，攻关3=③跨模态(文本×行情)融合。

### ① 招股书长文档解析（防幻觉抽取）= 版面解析 → 结构化分块 → 检索/推理

| 工具 | 类型 | 适配要点 | 链接 |
|---|---|---|---|
| **MinerU**(opendatalab) | 开源本地 | PDF→Markdown/JSON，版面+公式+表格，中英混排/扫描件友好，**最贴合且本地可跑** | [GitHub](https://github.com/opendatalab/MinerU) |
| **PP-StructureV3**(PaddleOCR) | 开源本地 | 复杂 PDF→Markdown/JSON 保结构，中文 OCR 强，表格/图表 | [GitHub](https://github.com/PaddlePaddle/PaddleOCR/blob/main/docs/version3.x/algorithm/PP-StructureV3/PP-StructureV3.md) |
| **Docling**(IBM) | 开源本地 | 表格数值/文本准确率 94%+，做表格交叉校验 | [基准 2025](https://procycons.com/en/blogs/pdf-data-extraction-benchmark/) |
| **PyMuPDF** | 开源底座 | 原生文本/坐标抽取，快速文本层+页码定位 | (库) |

> **防幻觉关键**（对应赛题书"抽取准确率≥80%/证据召回≥85%/可追踪率100%"）：表格/数字走**确定性管线**(MinerU/PP-Structure 抽取，不让 LLM 编数) + 每个风险结论强制**回链到原文片段**(证据 span)，正好满足"证据来源可追踪率 100%"。评测参考 [OmniDocBench (CVPR 2025)](https://github.com/opendatalab/OmniDocBench)。
> 分块/检索：数百页招股书纯长上下文会"Lost in the Middle"，**RAG + 结构感知分块**为主([RAGFlow 2025 综述](https://ragflow.io/blog/rag-review-2025-from-rag-to-context))；跨章节多跳(如"募资用途 vs 风险因素是否一致")用 **[GraphRAG (arXiv 2404.16130)](https://arxiv.org/abs/2404.16130) / [LightRAG (EMNLP 2025)](https://github.com/hkuds/lightrag)**（v1.5 起**原生集成 MinerU/Docling**，面向金融，与解析链无缝拼接，强烈推荐）。金融数值推理 [FinGEAR (arXiv 2509.12042)](https://arxiv.org/pdf/2509.12042)。
> 文档智能 SOTA(论文/创新点)：[LayoutLMv3 (arXiv 2204.08387)](https://arxiv.org/abs/2204.08387)(与你 CV 背景契合)、[DocLLM (arXiv 2401.00908)](https://arxiv.org/pdf/2401.00908)、长文档金融 QA 基准 [DocFinQA (ACL 2024, arXiv 2401.06915)](https://arxiv.org/abs/2401.06915)("长招股书 QA 是真难题"的好引用)。

### ② 多智能体协同（法务合规/财务穿透/市场分析 + 辩论查证）

| 框架 | 范式 | 适配 | 链接 |
|---|---|---|---|
| **LangGraph** | 图/状态机 | **首推**：planner-executor-critic 带状态/可回环/可人工检查点，正好实现赛题书"自主规划+辩论+查证链路" | [对比](https://www.datacamp.com/tutorial/crewai-vs-langgraph-vs-autogen) |
| AutoGen(微软) | 多Agent对话 | 适合"多专家辩论"(法务 vs 财务 vs 市场) | 同上 |
| CrewAI / MetaGPT | 角色/SOP | 上手快、固定流水线 | 同上 |

> 用 **LangGraph** 编排「**法务合规 Agent / 财务穿透 Agent / 市场分析 Agent + Critic(冲突辩论+证据查证)**」，直接映射赛题书要求；你的"多智能体/多供应商工作流"经验可迁移。
> **金融多智能体案例(强参考)**：**FinRobot**([GitHub](https://github.com/ai4finance-foundation/finrobot)｜[arXiv 2405.14767](https://arxiv.org/abs/2405.14767))；**FinGPT**([GitHub](https://github.com/AI4Finance-Foundation/FinGPT))做情感/事件抽取底座；**TradingAgents-CN**(**支持港股、后端已迁 FastAPI、多供应商模型路由**，与你栈完全一致，作工程脚手架)([GitHub](https://github.com/hsliuping/TradingAgents-CN)；原版 [arXiv 2412.20138](https://arxiv.org/pdf/2412.20138))。

### ③ 跨模态融合（基本面文本 × 市场情绪时序）+ 上市后表现验证
- 招股书基本面瑕疵(静态低频) + 发行期大盘/板块流动性(动态高频)融合 → 共振风险评分。
- 时序/异常：[FEDformer 金融时序异常+风险 (arXiv 2511.12951)](https://arxiv.org/abs/2511.12951)、[多元时序异常综述](https://pmc.ncbi.nlm.nih.gov/articles/PMC11723367/)、[ML 早期预警 (arXiv 2510.22287)](https://arxiv.org/pdf/2510.22287)；舆情/情绪 [事件感知情感因子 LLM (arXiv 2508.07408)](https://arxiv.org/html/2508.07408v1) / FinBERT。
- **验证口径(赛题书)**：用上市首日/5/20/60 交易日真实涨跌幅与破发标签反向验证，**重点优化"5 日内显著下跌"识别**(权重更高)——这应作为你模型调优与论文实验的主目标。

### 端到端架构（用东吴提供的数据落地）
```
东吴数据集(3-5年港股招股书PDF + 港股基本信息 + 历史行情OHLCV)
  → 解析: MinerU/PP-Structure + Docling 校验 → 结构化JSON(财务表/风险因素/对赌条款)，数字走确定性管线
  → 索引: LightRAG(GraphRAG) 实体/章节图 + 向量库
  → LangGraph 编排: Planner → [法务合规/财务穿透/市场分析 Agent] → Critic(辩论+证据查证, 可追踪率100%)
  → 共振风险引擎: 基本面暗雷(静态) × 发行期市场情绪(行情时序) → 风险评分 + 5日下跌预警 + 可解释报告(证据span)
  → FastAPI 服务/API(输入 公司名/代码/招股书 → 预警+报告)   ← 你已掌握 FastAPI
  → 用首日/5/20/60交易日真实跌幅破发数据回测验证
```
模型侧**多供应商路由**(DeepSeek/通义千问做中文金融；本地 RTX4070S 跑 OCR/向量化/FinBERT/时序)。

## 6. 推荐方案与主要风险（**风险已因"数据提供"而改写**）

**推荐(一句话)**：MinerU/PP-Structure 防幻觉解析(数字走确定性管线+证据span) + LightRAG 图检索 + LangGraph(法务/财务/市场 Agent + Critic 辩论查证) + 文本×行情跨模态共振 → FastAPI 原型/API，主攻"5 日内显著下跌"识别；骨架借鉴 FinRobot + TradingAgents-CN。

**主要风险**：
1. **金融专业性硬伤(现最高)**：对赌/赎回/破发/估值逻辑被金融评委看穿。对策：吃透赛题书+真实招股书，**强烈建议配 1 名金融/财会背景队友**。
2. **可运行性测试(硬门槛)**：指南明确初赛统一跑测试，**不能运行直接影响评审**；交付必须封装完整、依赖清晰、任意环境可部署。对策：从第一天就把"可一键运行 + Dockerfile/依赖声明"作为交付纪律。
3. **指标达标**：抽取准确率≥80%、证据召回≥85%、可追踪率 100% 是硬指标。对策：表格/数字确定性抽取 + 强制证据回链。
4. **财务数字幻觉**：Critic 强制数值校验，不让 LLM 编数。
5. **原创性声明**：缺交则初赛即出局——别漏。
6. ~~数据可得性~~ **已解除**：东吴提供 3-5 年招股书+基本信息+行情，无需自建(但需登录/向命题方领取数据集)。
7. **论文产出**：揭榜偏工程，可把"防幻觉证据可追踪解析""文本×行情共振 5 日下跌预警"做成可发表方法，对标 DocFinQA/FinGEAR 评测。
